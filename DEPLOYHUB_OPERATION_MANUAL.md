## 1. 系统概述

**DeployHub** 是一套面向内网环境的自动化部署服务平台，主要功能包括：

- **服务器管理**: 添加、编辑、删除服务器，支持密码和密钥两种认证方式
- **应用管理**: 管理 Java 应用配置，支持前后端分别打包
- **一键部署**: 支持多服务器并行部署，实时查看部署日志
- **脚本管理**: 可配置的部署脚本模板，支持变量替换
- **部署历史**: 完整的部署记录，支持回滚操作
- **系统设置**: 全局配置管理

### 系统架构

```
前端页面 (Vue 3)
    │
    ├── API请求 ──► 后端服务 (Spring Boot)
    │                      │
    │                      ├── MySQL 数据库
    │                      │
    │                      └── SSH连接 ──► 目标服务器群
    │
    └── WebSocket ──► 实时日志推送
```

## 3. 服务器管理

### 3.1 添加服务器

1. 点击左侧菜单 **服务器管理**
2. 点击右上角 **新增服务器** 按钮
3. 填写服务器信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 名称 | ✅ | 服务器名称（便于识别） |
| IP地址 | ✅ | 服务器 IP 地址 |
| 端口 | ❌ | SSH 端口，默认 22 |
| 用户名 | ✅ | SSH 登录用户名 |
| 认证方式 | ✅ | 密码 / 密钥 |
| 密码 | 条件必填 | 认证方式为"密码"时必填 |
| 密钥内容 | 条件必填 | 认证方式为"密钥"时必填 |
| 分组 | ❌ | 服务器分组名称 |
| 描述 | ❌ | 备注说明 |

4. 点击 **确定** 完成添加

### 3.2 编辑服务器

1. 在服务器列表中找到目标服务器
2. 点击操作列的 **编辑** 按钮
3. 修改需要变更的信息
4. 点击 **确定** 保存

### 3.3 删除服务器

1. 在服务器列表中找到目标服务器
2. 点击操作列的 **删除** 按钮
3. 在确认弹窗中点击 **确定**

> ⚠️ **注意**：删除服务器不会影响已有的部署记录

### 3.4 连通性测试

1. 在服务器列表中找到目标服务器
2. 点击操作列的 **测试连接** 按钮
3. 等待测试结果：
   - ✅ **在线**: 服务器连通正常
   - ❌ **离线**: 无法连接到服务器

> 💡 **提示**: 测试连接会自动更新服务器状态

### 4.2 编辑应用

1. 在应用列表中找到目标应用
2. 点击操作列的 **编辑** 按钮
3. 修改需要变更的信息
4. 点击 **确定** 保存

### 4.3 删除应用

1. 在应用列表中找到目标应用
2. 点击操作列的 **删除** 按钮
3. 在确认弹窗中点击 **确定**

### 4.4 构建操作

1. 在应用列表中找到目标应用
2. 点击操作列的 **构建** 按钮
3. 选择构建类型：
   - **构建前端**: 仅执行前端构建命令

> 💡 **提示**: 构建命令在应用所在服务器上执行，需确保构建工具（npm、mvn）已安装

## 5. 部署中心

### 5.1 选择应用和服务器

1. 点击左侧菜单 **部署中心**
2. 在左侧应用列表中选择要部署的应用
3. 在右侧服务器选择区域勾选目标服务器（可多选）

### 5.2 配置部署脚本

1. 在 **部署脚本** 输入框中填写部署命令
2. 支持使用变量替换，变量格式为 `${变量名}`

**常用变量列表：**

| 变量 | 说明 | 示例值 |
|------|------|--------|
| `${APP_NAME}` | 应用名称 | my-project |
| `${VERSION}` | 部署版本 | v1.0.0 |
| `${SERVER_HOST}` | 服务器地址 | 192.168.1.100 |
| `${SERVER_USER}` | 服务器用户 | deploy |
| `${TIMESTAMP}` | 时间戳 | 1715923200000 |

**脚本示例：**

```bash
#!/bin/bash

# 停止旧服务
pkill -f ${APP_NAME}.jar

# 复制新包
cp /tmp/${APP_NAME}-${VERSION}.jar /opt/apps/

# 启动新服务
nohup java -jar /opt/apps/${APP_NAME}-${VERSION}.jar > /var/log/${APP_NAME}.log 2>&1 &

echo "部署完成"
```

### 5.3 上传安装包（可选）

1. 点击 **上传安装包** 区域
2. 选择本地安装包文件（支持 JAR、WAR、ZIP）
3. 填写远程路径（目标服务器上的存放路径）
4. 点击 **上传**

### 5.4 触发部署

1. 填写版本号（可选）
2. 点击 **开始部署** 按钮
3. 在确认弹窗中点击 **确定**

### 5.5 实时日志查看

部署过程中：
1. 下方部署状态看板会显示每台服务器的部署进度
2. 点击任意服务器的 **查看日志** 按钮
3. 在弹出的终端窗口中实时查看部署日志

**部署状态说明：**

| 状态 | 图标 | 说明 |
|------|------|------|
| PENDING | ⏳ | 等待执行 |
| DEPLOYING | 🔄 | 部署中 |
| SUCCESS | ✅ | 部署成功 |
| FAILED | ❌ | 部署失败 |
| ROLLING_BACK | 🔙 | 回滚中 |
| ROLLED_BACK | ↩️ | 已回滚 |

## 9. 常见问题

### 9.1 服务器连接失败

**问题现象**: 测试连接显示离线，或部署时报 SSH 连接错误

**可能原因及解决方案**:

| 原因 | 解决方案 |
|------|----------|
| 网络不通 | 检查服务器 IP 是否正确，网络是否可达 |
| 端口未开放 | 确认服务器 SSH 端口（默认 22）已开放 |
| 用户名/密码错误 | 核对用户名和密码是否正确 |
| 密钥格式错误 | 确保密钥内容完整且格式正确 |
| 防火墙拦截 | 检查服务器防火墙是否允许连接 |

### 9.2 构建失败

**问题现象**: 点击构建后返回失败信息

**可能原因及解决方案**:

| 原因 | 解决方案 |
|------|----------|
| 目录不存在 | 检查配置的前端/后端目录是否存在 |
| 构建工具未安装 | 在目标服务器安装 npm、mvn 等工具 |
| 依赖下载失败 | 检查网络，配置镜像源 |
| 代码编译错误 | 修复代码中的编译错误 |

### 9.3 部署脚本执行失败

**问题现象**: 部署过程中脚本执行失败

**可能原因及解决方案**:

| 原因 | 解决方案 |
|------|----------|
| 脚本语法错误 | 检查脚本语法，在服务器上手动测试 |
| 权限不足 | 确保执行用户有足够权限 |
| 变量未定义 | 检查脚本中使用的变量是否正确 |
| 路径不存在 | 确认脚本中引用的路径存在 |

### 9.4 实时日志不显示

**问题现象**: 部署开始后日志终端没有输出

**可能原因及解决方案**:

| 原因 | 解决方案 |
|------|----------|
| WebSocket 连接失败 | 检查浏览器控制台是否有错误 |
| 网络延迟 | 稍等片刻，日志会自动推送 |
| 脚本无输出 | 检查脚本是否有输出语句 |

### 9.5 上传文件失败

**问题现象**: 上传安装包时报错

**可能原因及解决方案**:

| 原因 | 解决方案 |
|------|----------|
| 文件过大 | 检查文件大小，考虑分块上传 |
| 路径不存在 | 确认远程路径存在且有写入权限 |
| SSH 连接断开 | 检查服务器连接状态 |

## 11. 项目部署操作指南

本章节详细说明6个预置项目的部署操作步骤。所有项目均已在系统中预置应用配置和脚本模板。

### 前置条件

1. Docker 测试环境已启动（MySQL + 测试服务器）
2. DeployHub 后端服务已启动（端口 8086）
3. DeployHub 前端页面可访问（端口 6173+）
4. 已在「服务器管理」中添加测试服务器

### 添加测试服务器

1. 点击左侧菜单 **服务器管理**
2. 点击 **新增服务器**
3. 填写信息：

| 字段 | 值 |
|------|------|
| 名称 | 测试服务器 |
| IP地址 | localhost |
| 端口 | 2222 |
| 用户名 | root |
| 认证方式 | 密码 |
| 密码 | root123 |

> **注意**：DeployHub 后端运行在宿主机上，因此服务器地址使用 `localhost:2222`（映射到容器内的22端口）。如果 DeployHub 后端也运行在 Docker 网络内，则使用 `deployhub-test-server:22`。

4. 点击 **确定**，然后点击 **测试连接** 验证连通性

### 11.1 皇岗口岸合作查验综合管理系统

#### 11.1.1 后端部署

**应用信息：**

| 项目 | 值 |
|------|------|
| 应用名称 | 皇岗口岸-后端 |
| 应用类型 | BACKEND_ONLY |
| 项目路径 | D:/01-project/hg_yw |
| 构建命令 | mvn clean package -DskipTests |
| 产物路径 | ims-admin/target/ims-admin.jar |
| 远程部署路径 | /opt/ims |
| 服务端口 | 8011 |

**部署步骤（DEPLOY_ONLY 模式 - 上传JAR包部署）：**

1. 点击左侧菜单 **部署中心**
2. 在应用列表选择 **皇岗口岸-后端**
3. 在服务器列表勾选 **测试服务器**
4. 部署模式选择 **仅部署（DEPLOY_ONLY）**
5. 点击 **上传安装包**，选择本地构建好的 `ims-admin.jar` 文件
6. 远程路径填写 `/tmp`
7. 在脚本模板中选择 **后端JAR包部署**
8. 点击 **开始部署**
9. 在实时日志窗口中观察部署进度：
   - 停止旧服务
   - 创建目录
   - 部署JAR包
   - 检测Java路径
   - 启动服务（Docker环境使用nohup）
   - 清理临时文件
10. 部署完成后验证：


### 11.7 部署模式选择指南

| 场景 | 推荐模式 | 说明 |
|------|----------|------|
| 本地已构建好，只需上传部署 | DEPLOY_ONLY | 手动上传JAR/ZIP文件 |
| 从源码自动构建并部署 | FULL | 自动执行构建命令+传输+部署 |
| 仅构建不部署 | BUILD_ONLY | 在应用管理中点击构建按钮 |

### 11.8 脚本模板选择指南

| 脚本模板 | 适用场景 | 特点 |
|----------|----------|------|
| 后端JAR包部署 | 标准Spring Boot应用 | 自动检测systemd/nohup，创建服务 |
| 前端Dist包部署 | 前端静态资源 | 自动备份、解压、设权限、重载Nginx |
| 后端JAR包部署-带配置文件 | 需要外部配置文件的应用 | 支持--spring.config.additional-location |
| 后端JAR包部署-调度系统 | 调度系统专用 | 支持GoldenDB驱动、JarLauncher启动 |
